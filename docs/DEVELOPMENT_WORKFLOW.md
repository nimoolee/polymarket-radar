# 开发工作流

## 开始任务前

1. 阅读 `PROJECT_MEMORY.md`。
2. 阅读 `docs/DEVELOPMENT_WORKFLOW.md`。
3. 如果这两个文件不存在，先创建，再继续代码修改。

## 修改原则

1. 优先小步修改，不做无理由大重构。
2. Gamma 只作为市场发现来源；交易判断必须看 CLOB。
3. 涉及市场分类、时间解释、评分和策略逻辑的修改，必须同步更新 `PROJECT_MEMORY.md`。
4. 不允许用 mock 数据冒充真实数据。
5. 不允许默认自动下单。

## 完成任务后

1. 更新 `PROJECT_MEMORY.md`。
2. 说明改了哪些文件。
3. 说明核心逻辑、运行方式、测试方式和性能影响。
4. 如果 `PROJECT_MEMORY.md` 未更新，则任务未完成。
